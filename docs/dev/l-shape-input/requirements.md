# L字入力 要件

## 対象 Issue

- GitHub Issue: #29 L字型になぞる正解入力を許可する

## 背景

現在の入力は、縦または横の直線 3 ブロックだけを正解候補として扱う。遊びの幅を広げるため、隣接する 3 ブロックを L 字型になぞる入力も正解として扱いたい。

ただし、学習上の分かりやすさを保つため、正解保証とヒントは従来通り左から右、または上から下の直線候補に限定する。

## 要件

- プレイヤー入力として、3 ブロックの L 字型選択を許可する。
- L 字型でも、選択順の `a + b = c` または `a - b = c` が成立すれば正解にする。
- L 字型の引き算は、直線と同様に選択順を厳密に反映する。
- 斜め移動、飛び越し、同じセルの重複選択は引き続き無効にする。
- 通常の正解候補スキャンには L 字型を含めてよい。
- 正解保証数とヒント対象の算出には L 字型を含めない。
- 正解後に同じ 3 マスだけで構成される正解が残らないルールは、L 字型にも適用する。

## 非要件

- 4 ブロック以上の折れ線入力は扱わない。
- 斜め入力は許可しない。
- L 字型の保証配置は行わない。
- L 字型専用のヒント表示は作らない。

## 受け入れ条件

- L 字型の `5 -> 7 -> 12` が足し算として正解になる。
- L 字型の `9 -> 4 -> 5` が引き算として正解になる。
- 盤面生成と補充後の保証正解数は、直線方向の候補だけで判定される。
- ヒント対象は直線方向の候補だけになる。
- 既存の直線入力、逆方向入力、補充、ヒントが退行しない。
- `npm run lint`、`npm run build`、`npm test` が通る。
- In-App Browser で L 字操作を確認する。
