# 入力操作 要件

## 対象 issue

- #5 入力操作: マウス・タッチで縦横ドラッグ選択できるようにする

## 目的

PC とスマホの両方で、縦または横の連続した数字ブロックをドラッグ選択できるようにする。

## 要件

- Pointer Events を使い、マウスとタッチを共通処理にする。
- pointer capture 中でもセル解決が壊れないよう、DOM 接続では座標ベースの resolver を使えるようにする。
- ドラッグ開始セルを記録する。
- 2 個目のセルで縦方向または横方向をロックする。
- 最大 3 ブロックまで選択する。
- 斜め移動、飛び越し、途中で曲がる移動は選択に追加しない。
- 直前のセルへ戻った場合は 1 つ戻れる。
- pointer up で選択完了を通知する。
- pointer cancel で選択をリセットする。
- lost pointer capture でも選択をリセットする。
- pointer up では release 位置のセルを最終状態として反映する。
- 1 本目のドラッグ中に別 pointer が入っても、現在の選択を乗っ取らせない。
- controller は対象 root の `touch-action` を抑止し、破棄時に元へ戻す。
- 選択中の視覚表現は #6 の UI 実装で扱う。
- #5 では現在の placeholder 盤面に controller を接続し、選択中のブロックが視覚的に分かる最小表示まで実装する。
- #6 では、この入力を本番の盤面 UI と式プレビューへ接続する。

## 非スコープ

- 実際の盤面 DOM の描画。
- 選択した式の正解判定。
- 成功演出。

## 完了条件

- マウスとタッチの両方に使える Pointer Events ベースの controller がある。
- `clientX/clientY` と `elementFromPoint` を使ってセルを解決する helper がある。
- 純粋関数として、縦横の連続選択だけを拡張できる。
- テストで斜め、飛び越し、曲がり、最大 3 個制限を確認できる。
- テストで多指入力、pointer cancel、touch-action の復元を確認できる。
- テストで、最後の `pointermove` が間引かれても `pointerup` のセルを取り込めることを確認できる。
- テストで、戻り操作の最後の `pointermove` が間引かれても `pointerup` のセルで選択を縮められることを確認できる。
- テストで、lost pointer capture 後に入力が復旧することを確認できる。
- 初期画面の placeholder 盤面でドラッグ選択状態が見える。
