# 段階ヒント 要件

## 対象 issue

- #9 ヒント: 詰まりを支える段階ヒントを実装する

## 目的

プレイヤーが詰まったときに、答えを一気に出さず、探索を続けやすい手がかりを段階的に出す。

このゲームはスコア競争より、反射的に計算ができるようになる練習を目的にする。そのため、ヒントは罰ではなく、盤面へ視線を戻す支援として扱う。

## 要件

- 一定時間操作がない場合に段階的にヒントを進める。
- 第 1 段階では、正解候補に含まれる最初のブロックを薄く光らせる。
- 第 2 段階では、答えブロックを薄く光らせる。
- 第 3 段階では、正解ライン全体を薄く光らせる。
- 第 4 段階では、成立する式をプレビュー欄に表示する。
- ヒント対象は現在の盤面に実在する `allAnswers` の正解候補から選ぶ。
- 正解、選択、選択キャンセル、盤面更新でヒント状態をリセットする。
- 古い hint timer や queue 済み callback が、新しい状態にヒントを再表示しないようにする。
- ヒント表示は通常の選択状態、正解状態、消去状態と見分けられる。
- ヒント中も盤面操作は止めない。

## 非スコープ

- ヒント回数制限。
- ヒント利用によるスコア減点。
- 複数候補からの賢い候補選択。
- ヒント専用ボタン。

## 完了条件

- 無操作で段階ヒントが進む。
- ヒント対象セルが現在盤面の正解候補と一致する。
- 選択や盤面更新で古いヒント表示が消える。
- lint、build、test、my-review を通過する。
