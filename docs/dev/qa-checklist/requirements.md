# QA 整備 要件

## 対象 issue

- #10 QA: ロジックテストと手動確認シナリオを追加する

## 目的

盤面生成、正解判定、入力操作、GitHub Pages 配信が壊れていないことを確認できる状態にする。

## 要件

- `a + b = c` の主要ケースをテストで確認する。
- `a - b = c` の主要ケースをテストで確認する。
- 右から左、下から上の成立式も正解になることをテストで確認する。
- LV1 の保証正解が左から右、上から下に配置されることをテストで確認する。
- 生成される数字に `0` が含まれないことをテストで確認する。
- スマホ幅とデスクトップ幅の手動確認手順を repo 内に残す。
- GitHub Pages 上で初期画面と基本操作を確認する手順を repo 内に残す。
- README と AGENTS.md から QA 手順へ辿れるようにする。

## 非スコープ

- Playwright 等のブラウザ E2E 導入。
- CI 設定の追加。
- 端末実機の検証結果保存。

## 完了条件

- `npm test` で QA 回帰テストが実行される。
- `docs/qa/manual-checklist.md` に手動 QA が整理されている。
- `npm run lint`、`npm run build`、`npm test`、`my-review` を通過する。
