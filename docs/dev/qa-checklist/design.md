# QA 整備 設計

## 方針

自動テストでは、ゲームの壊れやすい契約を回帰テストとして固定する。

手動 QA では、静的配信、スマホ幅、デスクトップ幅、ドラッグ操作、演出、ヒントなど、DOM とブラウザ操作に依存する確認をチェックリスト化する。

## 自動テスト

`test/qa-regression.test.js` を追加する。

対象は以下。

- 足し算: `5 + 7 = 12` が正解。
- 引き算: `12 - 7 = 5` が正解。
- 右から左: 右端から左へなぞる順序で成立する式が正解。
- 下から上: 下から上へなぞる順序で成立する式が正解。
- LV1 保証方向: `generateBoard(1)` の `guaranteedAnswers` が最低 4 件あり、方向が左から右または上から下のみ。
- 0 非生成: 生成盤面の値がすべて `1..18` に収まる。

既存の個別テストと一部重複するが、issue #10 の受け入れ条件を読むための contract test としてまとめる。

## 手動 QA

`docs/qa/manual-checklist.md` を追加する。

確認観点は以下。

- ローカルで `npm run lint && npm run build && npm test` が通る。
- デスクトップ幅で初期表示、ドラッグ、正解演出、盤面更新、ヒントが確認できる。
- スマホ幅で表示崩れなく、縦横ドラッグができる。
- GitHub Pages で初期画面と基本操作が確認できる。
