# 正解判定 要件

## 対象 issue

- #4 正解判定: 3ブロックの足し算・引き算を全方向で判定する

## 目的

プレイヤーがドラッグで選んだ 3 ブロックが、レベル設定上の等式として成立するか判定する。

## 要件

- 選択順を式の順番として扱う。
- 最後のブロックを答えとして扱う。
- `a + b = c` を判定する。
- LV2 以降では `a - b = c` を判定する。
- LV1 から左から右、右から左、上から下、下から上の全方向を正解判定対象にする。
- 3 個以外の選択は不成立。
- 縦横の一直線でない選択は不成立。
- 連続していない選択は不成立。
- 引き算は選択順を厳密に反映する。

## 非スコープ

- DOM 入力イベント。
- 盤面表示。
- 正解後の消去。
- ヒント。

## 完了条件

- `5 -> 7 -> 12` が `5 + 7 = 12` として成立する。
- `9 -> 4 -> 5` が LV2 で `9 - 4 = 5` として成立する。
- 右から左、下から上の成立式も正解になる。
- `4 -> 9 -> 5` は `4 - 9 = 5` ではないため不成立になる。
