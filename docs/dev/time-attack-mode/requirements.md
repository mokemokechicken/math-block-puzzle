# 1分タイムアタック 要件

## 対象 Issue

- GitHub Issue: #30 1分タイムアタックのゲームモードを追加する

## 背景

通常モードは、学習・フラッシュカード的に一定数の正解を見つける体験になっている。別モードとして、1分間でテンポよく正解を重ね、スコアを伸ばすゲームモードを追加する。

## 要件

- 通常モードとは別に「1分アタック」モードを追加する。
- 制限時間は 60 秒。ただし、モード選択直後には countdown を開始しない。
- 現在選択中のレベル設定でタイムアタックを開始できる。
- 最初の1問を正解し、消えたブロックが補充された時点で countdown を開始する。
- 1問正解するごとに基本点 10 点を獲得する。
- 最初の1問も10点を獲得し、その後に60秒が始まる。
- 不正解では得点しない。
- 前回正解から 10 秒以内に次の正解を出すと倍率ポイントを獲得する。
- 倍率 `m(t)` は前回正解からの経過秒数 `t` で次の式により決める。

```text
m(t) = clamp(2 - ((t - 1) / 9), 1, 2)
clamp(x, 1, 2) = min(2, max(1, x))
```

- `t <= 1` は `2.0 倍`。
- `1 < t <= 10` は `2.0 倍` から `1.0 倍` まで線形に下がる。
- `t > 10` の場合、累積倍率ポイントを `1.0 倍` にリセットし、その正解は基本点のみとする。
- 累積倍率は `m(t) - 1.0` の上乗せ分を加算する。
- 追加点は `floor(基本点 * (累積倍率 - 1.0))` とする。
- その正解で得る点は `基本点 + 追加点` とする。
- 残り時間が 0 になったら入力を終了し、最終スコアを表示する。

## 非要件

- ハイスコア保存は行わない。
- ランキングや履歴は作らない。
- 通常モードの5問クリア条件は変更しない。
- タイムアタック専用レベルは作らない。

## 受け入れ条件

- 通常モードと1分アタックを切り替えられる。
- 1分アタック選択直後に残り時間、スコア、累積倍率が表示されるが、最初の正解補充までは countdown が進まない。
- 1問目の正解は10点になる。
- 1問目の正解ブロックが消えて補充された後に countdown が開始される。
- `t <= 1`、`t = 5.5`、`t = 10`、`t > 10` の境界でスコア計算が正しい。
- 端数が出る追加点は切り捨てられる。
- 時間切れでタイムアップ表示になり、最終スコアが表示される。
- 通常モードのレベル選択、5問クリア、ヒント、効果音が退行しない。
- `npm run lint`、`npm run build`、`npm test` が通る。
- In-App Browser でモード切替、1分アタック開始、正解によるスコア加算を確認する。
