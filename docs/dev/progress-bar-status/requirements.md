# 進捗バー表示 要件

## 対象 Issue

- GitHub Issue: #28 進捗表示をProgressバー化し、正解候補数表示を廃止する

## 背景

現在のゲーム状態には「盤面の正解候補数」として「N こ見つかる」が表示されている。これは探索の手掛かりとしては便利だが、フラッシュカード的に次々と見つける体験ではネタバレ感が強い。

ステージの進み具合は、数値だけでなく横長の Progress バーとして表示し、残りの見通しを視覚的に分かりやすくする。

## 要件

- 「N こ見つかる」の正解候補数表示を画面から削除する。
- 通常ステージの進捗を `correctCount / clearAnswerCount` に基づく Progress バーで表示する。
- Progress バーは現在値と最大値をアクセシブルに表現する。
- 数値の `0 / 5` などは残し、視覚とテキストの両方で進捗を確認できるようにする。
- レベル選択、正解後、リトライ、次のレベルで進捗表示が正しく更新される。
- デスクトップ幅、スマホ幅のどちらでも横にはみ出さない。

## 非要件

- クリア条件の変更はしない。
- 盤面生成や正解候補スキャンの内部ロジックは変更しない。
- タイムアタックなど別ゲームモードの UI はこの Issue では扱わない。

## 受け入れ条件

- 初期表示で Progress バーが `0 / 5` 相当を示す。
- 正解後に Progress バーが進む。
- 「こ見つかる」および正解候補数の UI が表示されない。
- 既存のレベル選択、効果音、ヒント、盤面補充が退行しない。
- `npm run lint`、`npm run build`、`npm test` が通る。
- In-App Browser でデスクトップ幅とスマホ幅を確認する。
