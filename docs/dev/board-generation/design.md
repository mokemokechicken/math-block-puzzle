# 盤面生成 設計

## 構成

`src/board.js` に盤面生成と候補スキャンを置く。

```text
MathBlockPuzzleBoard
├── createSeededRandom()
├── createEmptyBoard()
├── cloneBoard()
├── getLineCells()
├── listLinePlacements()
├── scanBoardForAnswers()
└── generateBoard()
```

## 生成手順

1. レベル設定から盤面サイズ、選択長、演算、保証配置方向、最低保証正解数を読む。
2. 空の盤面を作る。
3. 保証配置方向に沿って、空いている 3 マスの直線を選ぶ。
4. `a + b = c` または `a - b = c` の成立式を埋める。
5. 最低保証正解数のぶんだけ繰り返す。
6. 残りの空きマスを `1..18` で埋める。
7. 盤面をスキャンし、保証配置方向に最低数の正解があることを確認する。
8. 条件を満たさなければ再生成する。

## 正解候補スキャン

盤面生成には、生成結果を検証するための候補スキャンが必要になる。

このスキャンは、プレイヤー入力の妥当性判定とは別の責務として扱う。プレイヤー入力の連続性や選択状態は #4 以降で扱う。

## ランダム性

テストと再現性のため、seed 付きの軽量な疑似乱数を用意する。

本番では seed を省略し、実行時刻ベースの seed を使う。

## 保証正解と偶然正解

`guaranteedAnswers` は、難易度ごとの保証配置方向でスキャンした正解候補を指す。

`allAnswers` は、全ての正解判定方向でスキャンした正解候補を指す。

埋め込んだ式以外に偶然正解が発生することは許容する。プレイヤーにとっては発見の余地になるため。
