# 足し算・引き算ブロックパズル コンセプト

## 概要

小学校低学年向けの、足し算と引き算を自然に練習できる Web パズルゲーム。

数字のブロックが `w x h` の盤面に並び、プレイヤーはマウスまたはスマホのタッチ操作で、縦または横に連続したブロックをドラッグして選ぶ。選んだブロック列が等式として成立すると、ブロックが消えてスコアになる。

ゲームの目的は、計算問題を解かされている感覚ではなく、「数字の並びから成立する式を見つける」感覚で、足し算・引き算への慣れを作ること。

## 基本ルール

- 盤面には数字ブロックが格子状に並ぶ。
- プレイヤーは縦または横に連続するブロックをドラッグして選択する。
- 選択順がそのまま式の順番になる。
- 最後に選んだブロックを答えとして扱う。
- 最後以外のブロックを左辺として計算し、等式が成立したら成功。
- 成功したブロックは消える。
- 消えた場所には、上からブロックが落ちるか、新しいブロックが補充される。

## 入門モード

入門モードでは、選べるブロック数を 3 個までにする。

### 足し算

```text
2 -> 5 -> 7
```

この場合は `2 + 5 = 7` として成立する。

### 引き算

```text
9 -> 4 -> 5
```

この場合は `9 - 4 = 5` として成立する。

引き算は順番が重要なので、次の選び方は成立しない。

```text
4 -> 9 -> 5
```

`4 - 9 = 5` ではないため不成立。

## 発展モード

発展モードでは、4 ブロックの式を扱う。

### 3 項の足し算

```text
2 -> 3 -> 4 -> 9
```

`2 + 3 + 4 = 9` として成立する。

### 足し算と引き算の混合

```text
8 -> 3 -> 2 -> 9
```

`8 + 3 - 2 = 9` として成立する。

混合式は難易度が上がるため、入門モードの後に解放する。

## 難易度設計

### レベル 1

- 3 ブロックのみ。
- 足し算だけ。
- 答えは 10 以下。
- 盤面サイズは小さめ。例: `4 x 4`。

### レベル 2

- 3 ブロックのみ。
- 足し算と引き算。
- 答えは 10 以下。
- 盤面サイズは `5 x 5` 程度。

### レベル 3

- 4 ブロックを解放。
- `a + b + c = d` を扱う。
- 答えは 20 以下。

### レベル 4

- 4 ブロック。
- `a + b - c = d` や `a - b - c = d` を扱う。
- 順番を考える必要がある。

## UX 方針

- ドラッグ中に、選択中の式を画面上に表示する。
- 成功時は、成立した式を短く表示してからブロックを消す。
- 失敗時は強いペナルティを避ける。
- 低学年向けなので、制限時間よりも「いくつ式を見つけられるか」を中心にする。
- 計算問題の UI ではなく、パズルとして気持ちよく触れることを重視する。

## 盤面生成方針

完全ランダムにすると解ける式がない盤面になりやすいため、次の方針にする。

- 盤面生成時に、必ずいくつかの成立する式を埋め込む。
- 残りのマスはランダムな数字で埋める。
- プレイヤーが成功するたびに、新しい成立候補を補充する。
- 入門モードでは、常に 2 から 4 個程度の解ける式が盤面にある状態を目指す。

## 配布方針

Web アプリとして実装し、GitHub Pages で配布する。

初期実装では、静的ファイルだけで動作する構成を優先する。

- `index.html`
- `src/`
- `assets/`
- `docs/`

ビルドツールを使う場合も、最終成果物は GitHub Pages でホストできる静的アセットにする。

## 最初に作るもの

1. `5 x 5` の盤面。
2. 数字ブロックの表示。
3. マウスとタッチでの縦横ドラッグ選択。
4. 3 ブロックの `a + b = c` 判定。
5. 成功時の式表示とブロック消去。
6. 消えたブロックの補充。
