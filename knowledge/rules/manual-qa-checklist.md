# 手動 QA チェックリスト

## 実行前

- `npm run lint` が通る。
- `npm run build` が通る。
- `npm test` が通る。
- 変更内容が [現行実装の設計](../docs/design/current-implementation.md) と矛盾していない。

## デスクトップ幅

- 初期画面にタイトル、レベル選択、ゲームモード切替、効果音切り替え、レベル表示、進捗バー、数字盤面が表示される。
- レベル選択で別レベルに切り替えられる。
- ゲームモード切替で通常モードと1分アタックを切り替えられる。
- 数字ブロックが枠からはみ出さない。
- 横方向に 3 ブロックなぞると選択状態が付く。
- 縦方向に 3 ブロックなぞると選択状態が付く。
- L 字型に 3 ブロックなぞると選択状態が付き、式が成立すれば正解になる。
- 正解すると、選択ブロックが強調され、式が浮き上がる。
- 正解後、式が浮き上がっている間でも次の盤面をなぞり始められる。
- 正解後、ブロックが消える状態になり、正解した 3 マスが新しい数字へ更新される。
- 正解候補の保証回復が必要な場合だけ、他のブロックも最小限の範囲で更新される。
- 正解後、進捗バーが更新される。
- 正解後も、ヒント対象になる読みやすい方向の正解候補が維持される。
- 正解後、直前に消した 3 マスだけを同じ向きまたは逆向きになぞっても次の正解にならない。
- 正解を数回続けたとき、保証・ヒント対象の正解位置が直近の正解マスだけに偏り続けない。
- 5 問正解するとクリア表示になり、「もう一回」と「次のレベル」を操作できる。
- 1分アタックでは残り時間、スコア、倍率、直近獲得点が表示される。
- 1分アタックを選んだ直後は残り 60 秒のまま countdown が進まず、最初の正解ブロックが消えて補充された後に countdown が始まる。
- 1分アタック開始後、残り時間表示は約 1 秒単位で更新され、連続的に細かく再描画されない。
- 1分アタックでは正解時にスコアが加算され、通常モードの5問クリアでは終了しない。
- 1分アタックでは正解時に獲得点がレベル表示のステータス枠内へ表示され、10秒以内の連続正解で倍率が乗ったときは通常より目立つ表示と高めの正解音になる。
- 1分アタックで残り時間が0になるとタイムアップ表示になり、最終スコアが表示される。
- レベル選択、「もう一回」、「次のレベル」で新しいステージが始まると、盤面の下端まで見える位置へスクロールする。
- 不正解では盤面が更新されず、再挑戦できる。
- 30 秒以上無操作で段階ヒントが左から右、または上から下の候補に表示され、以後 5 秒ごとに進行して最終段階で式が表示される。
- 効果音切り替えでミュート状態を変更でき、ミュート時もゲーム進行が壊れない。
- リロード直後の最初のタップまたはドラッグ後、音声再生が許可される環境では正解音・不正解音が鳴る。

## スマホ幅

- 幅 390px 前後で、盤面と見出しが横にはみ出さない。
- レベル選択、ゲームモード切替、効果音切り替え、進捗バー、1分アタック状態表示が横にはみ出さない。
- 1 ブロックの数字が読み取れる。
- 指操作相当のドラッグで、横方向と縦方向の 3 ブロックを選択できる。
- 指操作相当のドラッグで、L 字型の 3 ブロックを選択できる。
- ページスクロールやテキスト選択がドラッグ操作を邪魔しない。
- 正解演出、盤面更新、ヒント表示が重なって読みにくくならない。

## 正解方向

- 左から右の `5 + 7 = 12` のような式が正解になる。
- レベル 7 で `1 + 6 = 7` のような、くり上がりではないが算数として正しい足し算も正解になる。
- 右から左になぞって成立する式も正解になる。
- 上から下になぞって成立する式が正解になる。
- 下から上になぞって成立する式も正解になる。
- L 字型になぞって成立する式も正解になる。
