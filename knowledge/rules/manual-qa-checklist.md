# 手動 QA チェックリスト

## 実行前

- `npm run lint` が通る。
- `npm run build` が通る。
- `npm test` が通る。
- 変更内容が [現行実装の設計](../docs/design/current-implementation.md) と矛盾していない。

## デスクトップ幅

- 初期画面にタイトル、レベル表示、正解候補数、数字盤面が表示される。
- 数字ブロックが枠からはみ出さない。
- 横方向に 3 ブロックなぞると選択状態が付く。
- 縦方向に 3 ブロックなぞると選択状態が付く。
- 正解すると、選択ブロックが強調され、式が浮き上がる。
- 正解後、ブロックが消える状態になり、新しい盤面へ更新される。
- 不正解では盤面が更新されず、再挑戦できる。
- 無操作で段階ヒントが表示され、最終段階で式が表示される。

## スマホ幅

- 幅 390px 前後で、盤面と見出しが横にはみ出さない。
- 1 ブロックの数字が読み取れる。
- 指操作相当のドラッグで、横方向と縦方向の 3 ブロックを選択できる。
- ページスクロールやテキスト選択がドラッグ操作を邪魔しない。
- 正解演出、盤面更新、ヒント表示が重なって読みにくくならない。

## 正解方向

- 左から右の `5 + 7 = 12` のような式が正解になる。
- 右から左になぞって成立する式も正解になる。
- 上から下になぞって成立する式が正解になる。
- 下から上になぞって成立する式も正解になる。
