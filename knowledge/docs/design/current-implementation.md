# 現行実装の設計

## 概要

このアプリは、GitHub Pages で配布できる静的 Web アプリとして実装する。`index.html` が `src/*.js` と `src/styles.css` を相対パスで読み込み、外部ビルド成果物を必要としない。

小学校低学年向けに、縦または横の 3 ブロックをなぞって `a + b = c` または `a - b = c` を見つける体験を中心にする。スコア競争や時間制限よりも、「見つけた」「つながった」「消えた」「式が見えた」の連続を優先する。

## ファイル構成

- `index.html`: GitHub Pages の root 配信と `file://` 直開きに対応する入口。
- `src/config.js`: 数字範囲、演算、方向、レベル設定。
- `src/rules.js`: 選択の妥当性と式の正解判定。
- `src/board.js`: 盤面生成、保証正解の配置、正解候補スキャン。
- `src/input.js`: Pointer Events によるドラッグ選択。
- `src/hints.js`: 段階ヒントの純粋ロジック。
- `src/audio.js`: Web Audio API による短い効果音とミュート制御。
- `src/main.js`: DOM 描画、入力接続、成功演出、盤面更新、ヒント controller。
- `src/styles.css`: レイアウト、盤面、選択、成功、消去、ヒント表示。
- `scripts/lint.js`: タブ文字と行末空白の簡易 lint。
- `scripts/build.js`: 静的配信に必要なファイルと script/link 参照の smoke check。
- `test/*.test.js`: Node.js test runner によるロジックと DOM 相当の回帰テスト。

JavaScript は classic script として `globalThis.MathBlockPuzzle*` に機能を公開する。読み込み順は `config`、`rules`、`board`、`input`、`hints`、`audio`、`main` の順にする。

## レベル設定

数字範囲はレベルごとに異なり、全体では `1..20` を扱う。`0` は生成しない。選択長は全レベルで 3。全レベルで 5 問正解するとクリアになる。

| レベル | 盤面 | 演算 | 保証・ヒントの学習ターゲット | 正解判定方向 | 保証配置方向 |
| --- | --- | --- | --- | --- | --- |
| 1: 5までのたし算 | `4 x 4` | 足し算 | `1..5`、答え 5 以下 | 全方向 | 左から右、上から下 |
| 2: 10までのたし算 | `4 x 4` | 足し算 | `1..10`、答え 10 以下 | 全方向 | 左から右、上から下 |
| 3: 10までのひき算 | `4 x 4` | 引き算 | `1..10`、答え 1 以上 | 全方向 | 左から右、上から下 |
| 4: 10までのたし算・ひき算 | `4 x 4` | 足し算、引き算 | `1..10` | 全方向 | 左から右、上から下 |
| 5: 20までのたし算 | `5 x 5` | 足し算 | `1..20`、答え 20 以下 | 全方向 | 左から右、上から下 |
| 6: 20までのひき算 | `5 x 5` | 引き算 | `1..20`、答え 1 以上 | 全方向 | 左から右、上から下 |
| 7: くり上がりのたし算 | `5 x 5` | 足し算 | 1 桁 + 1 桁、答え 11..18 | 全方向 | 左から右、上から下 |
| 8: くり下がりのひき算 | `5 x 5` | 引き算 | 11..18 から 1 桁を引き、答え 1 桁 | 全方向 | 左から右、上から下 |
| 9: たし算・ひき算マスター | `5 x 5` | 足し算、引き算 | `1..20` | 全方向 | 左から右、上から下 |

正解判定方向と保証配置方向は別概念として扱う。低レベルでも右から左、下から上の正解は受け付けるが、最低保証正解数を満たすための埋め込みは読みやすい方向に寄せる。

保証・ヒントの学習ターゲットと、プレイヤーの正解判定も別概念として扱う。例えばレベル 7 はくり上がりの足し算を保証・ヒント対象にするが、盤面上に `1 + 6 = 7` のような正しい足し算が見えている場合は正解として受け付ける。

## 正解判定

プレイヤーの選択順をそのまま式の順番として扱い、最後のブロックを答えにする。

判定手順:

1. 選択数がレベル設定の `selectionLength` と一致するか確認する。
2. 各セルに `row`、`col`、`value` があるか確認する。
3. 1 個目から 2 個目の差分が縦横 1 マスの方向か確認する。
4. 3 個目以降も同じ差分で連続しているか確認する。
5. 方向がレベルの `validationDirections` に含まれるか確認する。
6. 値の並びが `a + b = c` または `a - b = c` として成立するか確認する。
7. 成立式が現在レベルの演算種別と正解判定用の数字範囲を満たすか確認する。くり上がり・くり下がり条件は保証・ヒント対象の絞り込みに使い、見えている正しい式を無効にはしない。

引き算は選択順を厳密に反映する。例えば `9 -> 4 -> 5` は成立するが、`4 -> 9 -> 5` は成立しない。

## 盤面生成

`generateBoard()` はレベル設定に基づいて空盤面を作り、保証配置方向に沿って成立式を埋め込む。残りのセルはレベルごとの数字範囲で埋める。

生成手順:

1. レベル設定から盤面サイズ、選択長、演算、保証配置方向、最低保証正解数を読む。
2. 空の盤面を作る。
3. 保証配置方向に沿って、空いている 3 マスの直線を選ぶ。
4. `a + b = c` または `a - b = c` の成立式を埋める。
5. 最低保証正解数のぶんだけ繰り返す。
6. 残りの空きマスをレベルごとの数字範囲で埋める。
7. 盤面をスキャンし、保証配置方向に最低数の正解があることを確認する。
8. 条件を満たさなければ再生成する。

盤面スキャンは `rules.evaluateEquation()` と同じ式評価を使う。通常の正解候補スキャンではプレイヤーが正解できる式を列挙し、保証・ヒント用のスキャンでは学習ターゲットに合う式だけへ絞る。保証正解以外に偶然正解が発生することは許容する。

## 入力

Pointer Events を使い、マウスとタッチを共通処理にする。

- 1 個目はどのセルでも選択できる。
- 2 個目は上下左右に隣接している必要がある。
- 2 個目で方向をロックする。
- 3 個目以降は同じ方向に 1 マスずつ進む場合だけ追加する。
- 最大 3 個まで選択する。
- 直前のセルに戻った場合は最後のセルを取り消す。
- 斜め、飛び越し、途中で曲がる動きは無視する。
- `pointerup` では release 位置のセルを反映してから完了 callback を呼ぶ。
- active pointer が存在するとき、別の pointer は現在の選択を乗っ取れない。
- controller 初期化時に root の `touch-action` と `user-select` を抑止し、`destroy()` で元へ戻す。

DOM 接続では `clientX/clientY` と `elementFromPoint()` を使い、pointer capture 中でもセル解決が壊れないようにする。

## UI とゲーム進行

初期表示ではレベル選択、効果音切り替え、レベル名、状態表示、正解数、正解候補数、式プレビュー、数字盤面を表示する。現在の既定レベルは `src/main.js` の `createGameState()` でレベル 1。

式プレビュー:

- 0 個: `ブロックをなぞって式を作ろう`
- 1 個: 選択値だけ表示
- 2 個: `a ± b = ?`。足し算のみのレベルでは `a + b = ?`
- 3 個で成立: `a + b = c` または `a - b = c`
- 3 個で不成立: `a ? b = c`

正解時は、選択セルに成功状態を付け、選択セルの中心位置から成立式を浮き上がらせる。その後、選択セルに消去状態を付け、成功演出後に選択された 3 マスを同じ位置で新しい数字へ補充する。補充後、消した 3 マスだけで構成される正解候補は同じ向きでも逆向きでも必ず壊し、同じ場所をなぞるだけでは次の正解にならないようにする。補充後に読みやすい方向の保証正解数が規定より少ない場合は、元盤面に存在し、消去セルに触れていない正解候補を壊さない範囲で他セルも最小限置換し、保証正解数の回復を試みる。補充後は盤面を再スキャンし、正解候補数を更新する。正解候補が 0 になった場合、または 5 問正解した場合はクリア状態にする。

盤面更新では refresh token と timer を使い、古い callback が後続盤面を上書きしないようにする。再描画時には既存の入力 controller とヒント controller を破棄する。

`prefers-reduced-motion: reduce` では成功演出と cleanup timer を短縮する。ただし、成功フィードバックとして認識できる短い表示時間は残す。

## 段階ヒント

ヒントは現在の盤面に存在する `guaranteedAnswers` の先頭候補を対象にする。つまり、ヒントは必ず左から右、または上から下の読みやすい方向を示す。右から左、下から上の正解判定は受け付けるが、ヒント対象にはしない。賢い候補選択やヒント回数制限、スコア減点は現行スコープ外。

段階:

1. `source`: 正解候補の最初のブロックを薄く光らせる。
2. `answer`: 正解候補の最後のブロックを答えとして光らせる。
3. `line`: 正解候補の 3 ブロック全体を光らせる。
4. `expression`: line を維持し、プレビュー欄に式を表示する。

既定の遅延は 6 秒、10 秒、14 秒、18 秒。正解、選択、選択キャンセル、盤面更新ではヒント状態をリセットする。hint token と timer 解除により、古い callback が新しい状態へヒントを再表示しないようにする。

## 効果音

効果音は `src/audio.js` で Web Audio API を使って合成する。外部音源ファイルは使わない。

- 正解: 短い上昇音。
- 不正解: 控えめな低音。
- クリア: 短いジングル。
- ヒント: 控えめな短音。

AudioContext はユーザー操作後に作成または再開する。モバイルブラウザでは autoplay 制約が強いため、盤面の `pointerdown` 時点で短い無音トーンを開始して音声をアンロックし、suspended context では resume 完了後の現在時刻で効果音を予約する。Web Audio API が使えない環境、または autoplay 制約で再生できない環境でも、音を鳴らさずゲーム進行を継続する。UI にはミュート切り替えを表示する。

## 検証

通常の変更では次を実行する:

```sh
npm run lint
npm run build
npm test
```

表示や操作に関わる変更では、[手動 QA チェックリスト](../../rules/manual-qa-checklist.md) の該当項目も確認する。
